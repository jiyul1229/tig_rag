## 💡 RAG 시스템 데이터 적재 및 검색 가이드

RAG 담당자가 제공된 용접 기술 문서와 동영상 메타데이터를 활용하여 검색 파이프라인을 효율적으로 구축할 수 있도록 돕는 데이터 활용 요령입니다.

### 1. 지식 데이터베이스 적재 및 검색 (텍스트 중심)
* **적재 대상 문서:** `1_용접_기술_가이드_문서/` 폴더 내의 Markdown(.md) 파일 12종.
* **청킹(Chunking) 전략:**
  - 본 가이드 문서는 현장감 넘치는 Q&A 단락과 숙련공의 팁(`Secret Tip`)으로 명확히 구획화되어 있습니다.
  - 마크다운의 헤더(예: `### Q1. ...`, `### A1. ...`) 단위로 문서를 분할하여 개별 척(Chunk)으로 구성하고 VectorDB에 적재할 것을 권장합니다.
  - 이로써 질문과 대답의 문맥적 연관성을 높이고 RAG의 검색 정밀도를 최대화할 수 있습니다.
* **표준 용어와 현업 은어 병행 매핑:**
  - 문서 내에 `이면 비드(백비드)`, `가용접(테크용접)`, `스테인리스강(서스)` 처럼 표준 용어와 현업 실무 용어가 함께 쓰이고 있어, 작업자의 비정형 질문에도 높은 의미론적 유사도(Semantic Similarity) 검색 매칭이 일어납니다.

### 2. 동영상 메타데이터 JSON 활용 및 텍스트 매핑
* **활용 대상 데이터:** `2_동영상_구간_메타데이터_JSON/` 폴더 내의 JSON 파일 11종.
* **비디오 세그먼트 검색 방식:**
  - 각 JSON 파일은 비디오의 전체 내용이 아닌 **1~2분 단위의 핵심 동작 구간(Segments)**으로 쪼개져 있습니다.
  - 각 세그먼트의 `keywords`(키워드), `description`(구간 설명), `transcript`(자막 대본) 데이터를 텍스트 데이터베이스 검색 영역에 추가하거나 별도의 메타데이터 필드로 연동합니다.
  - 사용자가 특정 용접 테크닉이나 실무 요령을 질문할 때, 텍스트 문서뿐만 아니라 **가장 연관성이 높은 동영상 세그먼트**를 점수로 환산하여 매핑할 수 있습니다.
* **영상 매핑 변수 추출:**
  - 검색 결과로 가장 점수가 높은 비디오의 `video_id`, 해당 세그먼트의 `start_time` 및 `end_time`을 추출하여 시스템 상에서 연결된 영상(`3_시연_동영상_원본/` 폴더의 대응되는 MP4 파일)의 해당 시간대를 유저에게 링크 또는 조회 수단으로 제공합니다.

### 3. LLM 기술 답변 생성 시 프롬프트 제어
* **핵심 원칙:** 답변의 주체는 항상 명확한 **기술 답변 텍스트**여야 하며, 영상은 단순히 보완적인 참고 용도로만 기능하도록 설계합니다.
* **시스템 인스트럭션(System Instruction) 설정:**
  - LLM(Gemini 등)이 답변을 생성할 때 컨텍스트로 입력된 비디오 정보에 현혹되어 "영상에서 볼 수 있듯이", "02:54 구간을 참고하여" 같은 표현을 생성하지 않도록 강제해야 합니다.
  - LLM 호출 환경에서 아래와 같은 페르소나 및 주의 규칙을 설정하십시오:
    > "제공된 기술 문서 및 비디오의 팩트 지식만을 토대로 질문에 대한 기술 답변을 명확하게 작성하십시오. 답변 본문 내에는 '영상', '비디오', '유튜브', '타임스탬프', '몇 분 몇 초', '구간' 등 동영상 자료와 관련된 어떠한 단어나 시간 언급도 절대로 포함하지 마십시오. 비디오의 자막 및 설명 정보는 순수하게 핵심 기술 사실로만 필터링하여 완결된 일반 설명글로 서술해야 합니다."

## 📈 데이터셋 구축에 따른 시너지 효과
* 본 정제 패키지를 RAG 엔진(예: ChromaDB, Pinecone, LangChain 등)에 연동하면, 사용자의 용접 현장 질의에 대해 **정확하고 완결된 가이드라인 텍스트 답변이 기본 제공**되고, 동시에 **해당 기술이 실제 시연되는 1분 분량의 동영상 구간**을 직관적으로 연동해 주는 강력한 대화형 교육 데이터 파이프라인을 완성할 수 있습니다.
